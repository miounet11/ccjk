OpenClaw 任务执行方法论 v1.0（2026-03-07 版）

核心原则（不可违背的四条铁律）

1. 最小能力原则
永远先尝试用最轻、最确定、最可控的能力解决问题。只有当前能力明确不足时，才升级到下一层。
2. 可量化决策原则
每一次工具/能力升级，都必须回答三个量化问题：
  • 预期提升幅度（时间/质量/成功率）至少多少 %？
  • 失败成本 / 协调成本是多少？
  • 如果失败，我有明确的回滚 / 降级路径吗？
3. 可观测闭环原则
每种能力使用后，必须记录：
  • 用了什么
  • 实际效果打分（1–10）
  • 是否值得下次继续用 / 升级 / 放弃
4. 反熵原则
多 agent / 多线程永远是增加熵的行为。只有当任务本身熵很高（不确定性大、分工天然明显）时，才允许引入；否则优先降熵（简化）。
