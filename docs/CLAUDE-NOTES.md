# Claude Code Notes

Durable architecture/gotchas for AI coding agents working in this repo. Not regenerated by `/init` ‚Äî edit by hand.

`CLAUDE.md` and `clavue.md` at the repo root are rewritten by Clavue's `/init`. Reference this file from there so the content survives regeneration.

## Build, Test, Run

Node 20+, pnpm 10.17.x. The repo is a pnpm workspace; the root package (`ccjk`) depends on `@ccjk/wire` and `@ccjk/evolution` being built first, which is why `build` and `typecheck` chain through `build:release:deps`.

- `pnpm dev` ‚Äî run CLI from source via `tsx src/cli.ts`
- `pnpm build` ‚Äî `pnpm --filter @ccjk/wire build && pnpm --filter @ccjk/evolution build` then `unbuild` at root ‚Üí `dist/`
- `pnpm start` ‚Äî run built CLI via `bin/ccjk.mjs` (which imports `dist/cli.mjs`)
- `pnpm typecheck` ‚Äî builds wire/evolution first, then `tsc --noEmit` at root
- `pnpm lint` / `pnpm lint:fix` ‚Äî ESLint flat config (`eslint.config.ts`)
- `pnpm format` ‚Äî prettier on `src/**/*.ts`

Tests use Vitest with three configs:
- `pnpm test:run` ‚Äî main unit suite (`vitest.config.ts`, `pool: forks` + `singleFork: true`, 80% coverage thresholds)
- `pnpm test:integration:run` ‚Äî `vitest.integration.config.ts`; expects local Postgres/Redis/ES (see `env` block), 70% thresholds
- `pnpm test:e2e:run` ‚Äî `vitest.e2e.config.ts`; CI-style env (`FORCE_COLOR=0`, `retry: 2`, `bail: 1`), 60% thresholds
- Single test: `pnpm vitest run path/to/file.test.ts` or `pnpm vitest run -t "test name"`
- `pnpm test:release` ‚Äî curated subset run before publishing (see `package.json` for the exact list)

Release gate: `pnpm release:verify` (`scripts/release-verify.mjs`). Full gate: `pnpm release:verify:full`. `prepublishOnly` runs `validate-prepublish.mjs` + `contract:check` + `build` ‚Äî do not skip.

## Architecture

### Workspace layout

Root package `ccjk` is the user-facing CLI (npm: `ccjk`, bin: `bin/ccjk.mjs` ‚Üí `dist/cli.mjs`). Workspace packages under `packages/`:

- `@ccjk/wire` ‚Äî shared Zod schemas / message types for remote control. Pure types, no runtime side effects.
- `@ccjk/daemon` ‚Äî background daemon for `ccjk remote`. Reads `~/.ccjk/daemon.json`. Built with `pkgroll`.
- `@ccjk/server` ‚Äî Fastify + Prisma backend for remote-control web/app sessions.
- `@ccjk/evolution` ‚Äî depends on root build chain; built before the root bundle.
- `@ccjk/app`, `@ccjk/telegram-bot` ‚Äî additional surfaces.

`pnpm-workspace.yaml` uses `catalog:` deps ‚Äî when adding a dep, add it to the workspace catalog there, not piecemeal per package.

### CLI startup (lazy-load)

`src/cli.ts` ‚Üí `src/cli-lazy.ts` ‚Üí `src/cli-commands.ts`. The startup path classifies argv into kinds (`interactive | slash | help | version | command | quick-provider | plain-args`) and dynamically `import()`s only the command module needed. Don't add eager top-level imports of heavy command files in `cli-lazy.ts` or you regress cold-start time.

`src/cli-helpers.ts` owns: cloud-bootstrap worker fork, startup spinner, command-discovery banner, quick-provider launch, health-alerts check. These are called from `cli-lazy.ts`'s startup orchestration.

### Build / bundling rules (`build.config.ts`)

Critical: `unbuild` is configured to **inline all npm dependencies** into `dist/cli.mjs`. The `rollup:options` hook overrides `external` to force-bundle anything that isn't a Node builtin or one of the explicit native externals (`fdir`, `tinyglobby`, `web-tree-sitter`, `tar`, `ioredis`, `better-sqlite3`, `globby`, `unicorn-magic`). Reason: `npm install -g ccjk` does not reliably install transitive deps, so they must be bundled.

When adding a dep with native bindings, add it to **both** `externals` and `allowedExternals` in `build.config.ts`. The `build:done` hook copies `src/i18n/locales/**/*.json` and `templates/**` into `dist/` ‚Äî preserve this if editing the build hook.

### Menu / init flow (highest-churn area)

This is the most error-prone surface. Key invariants:

- `src/commands/menu.ts` (option dispatch) must call:
  - Option 1 ‚Üí `init({ skipBanner: true })` (full init), **not** `simplifiedInit()`
  - Option 1 must pass `codeType: 'claude-code'` to override stale `~/.ccjk/config.toml` from prior installs
  - Options 3/4 ‚Üí `configureApiFeature()` / `configureMcpFeature()` from `src/utils/features.ts`
  - Options 5‚Äì7 ‚Üí `configureDefaultModelFeature` / `configureAiMemoryFeature` / `configureEnvPermissionFeature`
  - Utility entries (0/S/-/+/D/H/R) return `undefined` to skip the "return to menu" prompt
- ccjk's own config lives at `~/.ccjk/config.toml` (controlled by `ZCF_CONFIG_DIR`), **separate from** `~/.claude/`. A full Claude uninstall leaves stale `~/.ccjk/config.toml`; `resolveCodeType()` in `src/utils/code-type-resolver.ts` is the central resolver (stored config ‚Üí fresh detection ‚Üí default). Filesystem detection in `detectCodeToolType()` lives in `src/utils/auto-config/smart-defaults.ts`.
- `auto.mcp` default of `0` is set in `src/config/migrator.ts` ‚Äî not in `smart-defaults.ts`. The MCP gatekeeper hook is disabled by default (it forked bash+node on every MCP call).

Smoke test for menu wiring: `echo '<option>' | node dist/cli.mjs 2>&1`.

### Claude Code config-manager rules

`src/utils/claude-code-config-manager.ts` owns merging into `~/.claude/settings.json`. Priority gotcha: in Claude Code itself, `settings.model` overrides `settings.env.ANTHROPIC_*`. When ccjk writes per-tier env vars (`ANTHROPIC_MODEL`, `ANTHROPIC_DEFAULT_HAIKU_MODEL`, ‚Ä¶), the manager must **delete** `settings.model` so the env vars take effect ‚Äî otherwise per-context model selection silently breaks. Regression coverage: `tests/utils/claude-code-config-manager.test.ts` and `tests/integration/model-priority.test.ts`.

### Anti-aggression principle

ccjk complements Claude Code; it does not auto-run things in Claude's runtime. Skills only run on explicit user invocation. Don't add background hooks that fire on every tool call. Skill files are **not** vendored in this repo ‚Äî they come from the `superpowers` plugin at `~/.claude/plugins/superpowers/skills/` at runtime.

### Generated rule baseline

`clavue.md`, `AGENTS.md`, and the root `CLAUDE.md` are owned by Clavue's `/init` and are rewritten from local repo signals. Put durable guidance here in `docs/` instead.

## Tooling Gotchas

- macOS `cat` has no `-A`; use `od -c` or `hexdump`.
- zsh doesn't expand `*.md` globs through the Bash tool ‚Äî quote patterns or use `python3 -c "import glob; ‚Ä¶"`.
- File edits can appear successful but be reverted by IDE/file watchers between tool calls; verify with `grep`/`Read` after writing, or chain `sed && git add` in one command.
- Parallel tool calls: a sibling failure can fail the whole batch ‚Äî prefer sequential when one call's success is load-bearing for the next.
