# BMad アジャイルフロー

短いサイクルで計画→実行→検証→改善を回すためのワークフローです。少人数のスプリントや PoC に向いています。

## ステップ概要

1. **Background**：現状・課題・制約を整理  
2. **Mission**：今回のゴール、成功指標  
3. **Action**：具体的タスクと担当/優先度  
4. **Deliver**：成果物、レビュー観点、リリース手順  
5. **Retro**：振り返り、次サイクルへの改善

## 使い方

```
/ccjk:bmad-init Improve onboarding speed for new developers
```

オプションで期間、人数、依存サービスなどを付けると計画精度が上がります。

## ベストプラクティス

- 小さいスコープで回し、成果をすぐに共有する  
- Retro で見つかった改善を次サイクルの Mission/Action に反映  
- `/git-worktree` と組み合わせて複数案を短時間で試し、最良案を採用する
