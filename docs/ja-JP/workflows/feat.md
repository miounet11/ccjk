# Feat ワークフロー

小～中規模の機能開発を素早く計画するためのテンプレートです。ユーザーストーリーからタスク分割、テストまでをまとめて生成します。

## 出力内容

- 背景・目標・非機能要件の整理  
- ユーザーストーリーと受入基準  
- 仕様サマリと UI/UX 要点（必要な場合）  
- タスク分割と優先度  
- テスト計画・リスクとトレードオフ

## 使い方

```
/ccjk:feat Add user comment feature
```

追加すると良い情報：
- スコープの上限（例：今回は表示のみ、投稿は別スプリント）  
- 既存 API/DB モデルの制約  
- パフォーマンスやセキュリティ要求

## ベストプラクティス

- 生成結果を Issue / PR 説明にそのまま貼り付けて合意形成を早める  
- `/ccjk:workflow` と組み合わせて詳細設計やテスト計画を発展させる  
- 大きな機能は複数の Feat に分け、優先度順に着手する
