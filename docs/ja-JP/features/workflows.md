# ワークフローシステム

CCJK は開発作業を構造化する複数のワークフローを同梱し、`ccjk init` で自動導入します。

## 主なワークフロー

- **6 段階ワークフロー**：調査/設計/実装/テスト/ドキュメント/振り返りのフロー  
- **Feat ワークフロー**：機能開発の計画と落とし込み  
- **BMad**：短サイクルのアジャイル実践  
- **Spec ワークフロー**：仕様作成〜実装までの統合  
- **Git スマートコマンド**：`/git-commit` などの自動化プロンプト

## 使い方

- インタラクティブ導入：`npx ccjk` → 2 (Import Workflows)  
- CLI：`npx ccjk init -s --workflows all` または `--workflows skip`  
- 会話中：`/ccjk:workflow <内容>`、`/ccjk:feat <機能名>`、`/git-commit` など

## ベストプラクティス

- プロジェクトでよく使うフローだけ残し、不要なものは `workflows/custom/` に整理  
- `/git-worktree` と併用して複数案を並行生成  
- 出力スタイルや言語設定と組み合わせて、チーム標準に合わせる
