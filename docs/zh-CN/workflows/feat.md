# 功能开发工作流

功能开发工作流面向新功能设计与实现，预置了专业的规划师（Planner）与 UI/UX 设计师（UI/UX Designer）代理，帮助您系统化地完成从需求分析到实施计划的完整流程。

## 功能概述

功能开发工作流专注于新功能的完整开发周期，通过结构化的阶段流程，确保功能开发的质量和完整性。

### 核心特点

- 🎯 **需求驱动**：从需求分析开始，确保功能符合用户期望
- 📐 **多方案设计**：生成多个可行方案并评估优劣
- 🎨 **UI/UX 支持**：提供专业的 UI/UX 设计建议
- 📋 **详细计划**：输出可执行的任务分解和实施计划
- 🤖 **专业代理**：内置规划师和 UI/UX 设计师代理

## 使用方法

### Claude Code

在 Claude Code 中使用以下命令：

```
/ccjk:feat <功能描述>
```

**示例**：
```
/ccjk:feat 实现用户评论功能，支持点赞、回复和删除
```

### Codex

> ⚠️ **注意**：功能开发工作流（feat）目前仅在 Claude Code 中提供，Codex 暂不支持此工作流。

如果需要类似功能，可以在 Codex 中使用 `/prompts:workflow` 六阶段工作流来实现功能开发任务。

## 工作流阶段

功能开发工作流包含四个核心阶段：

### 1. 需求确认

**目标**：深入理解功能需求和业务场景

**AI 行为**：
- 分析功能描述和用户需求
- 识别功能范围和边界
- 确定成功标准和验收条件
- 理解业务价值和用户场景

**输出**：
- 功能需求文档
- 用户故事
- 验收标准
- 成功指标

**示例输出**：
- 功能范围：用户评论、点赞、回复、删除
- 用户角色：普通用户、管理员
- 验收标准：用户可以发布评论、点赞他人评论、回复评论、删除自己的评论

### 2. 方案规划

**目标**：生成多个可行的技术方案并评估

**AI 行为**：
- 提出 2-3 个不同的技术方案
- 评估各方案的技术复杂度
- 分析方案的可扩展性和维护性
- 考虑性能、安全性和成本因素
- 推荐最适合的方案

**输出**：
- 多个技术方案
- 方案对比分析
- 技术选型建议
- 架构设计草图

**示例输出**：
- 方案 A：使用现有数据库表结构，简单快速
- 方案 B：引入评论树结构，支持无限嵌套回复
- 方案 C：使用图数据库，支持复杂关系查询
- 推荐：方案 B（平衡了复杂度和功能完整性）

### 3. UI/UX 支撑

**目标**：提供专业的用户界面和交互设计建议

**AI 行为**：
- 设计用户界面布局
- 提供交互流程建议
- 生成线框图和原型建议
- 编写界面文案和提示语
- 考虑用户体验和易用性

**输出**：
- UI 设计建议
- 交互流程图
- 界面线框图描述
- 文案和提示语
- 响应式设计考虑

**示例输出**：
- 评论列表布局：时间倒序、头像+昵称+内容
- 交互流程：点击回复 → 显示回复框 → 输入内容 → 提交
- 文案："发表评论"、"回复"、"点赞（123）"

### 4. 实施计划

**目标**：输出详细的任务分解和实施计划

**AI 行为**：
- 将功能拆解为具体开发任务
- 确定技术实现细节
- 制定开发顺序和依赖关系
- 估算开发工作量
- 提供测试策略

**输出**：
- 详细任务列表
- 技术实现细节
- 开发时间估算
- 测试计划
- 部署考虑

**示例输出**：
1. 创建评论数据模型（2 小时）
2. 实现评论 API 接口（4 小时）
3. 开发前端评论组件（3 小时）
4. 实现点赞功能（2 小时）
5. 实现回复功能（3 小时）
6. 实现删除功能（1 小时）
7. 编写测试用例（2 小时）
8. 性能优化（1 小时）

## 使用场景

### 场景 1：新功能开发

```
/ccjk:feat 添加购物车功能，支持多商品、数量修改和批量结算
```

适合需要完整规划的新功能，从需求到实施的完整流程。

### 场景 2：功能增强

```
/ccjk:feat 增强用户个人中心，添加数据统计和活动历史展示
```

适合在现有功能基础上的增强和扩展。

### 场景 3：复杂业务逻辑

```
/ccjk:feat 实现订单系统，支持多种支付方式、优惠券和物流跟踪
```

适合包含复杂业务逻辑的功能，需要仔细规划和技术选型。

## 最佳实践

### 1. 需求描述要清晰

提供清晰、详细的功能描述可以获得更好的规划结果：

```
# 好的描述
/ccjk:feat 实现用户评论功能，支持文本、图片、点赞、回复、删除，管理员可以审核和删除任何评论

# 不够清晰的描述
/ccjk:feat 评论功能
```

### 2. 结合项目上下文

在执行功能开发工作流前，建议先初始化项目配置：

```bash
# 1. 初始化项目（如果还没做）
/init-project

# 2. 执行功能开发工作流
/ccjk:feat <功能描述>
```

这样可以确保功能设计与项目整体架构一致。

### 3. 规划完成后执行

功能开发工作流的输出可以直接用于后续开发：

```bash
# 1. 使用功能开发工作流规划
/ccjk:feat 实现用户评论功能

# 2. 根据规划结果，使用六阶段工作流实施
/ccjk:workflow 实现评论的数据模型和 API 接口

# 3. 继续实施其他任务
/ccjk:workflow 实现评论的前端组件和交互

# 4. 使用 Git 工作流提交
/git-commit
```

### 4. 与其他工具结合

将规划结果同步到项目管理工具：

- 📋 **Issue 追踪**：将任务分解同步到 GitHub Issues 或 Jira
- 📝 **文档管理**：将方案设计保存到项目文档
- 👥 **团队协作**：与团队成员分享规划结果
- 🔄 **迭代管理**：将功能规划纳入迭代计划

### 5. 持续迭代

功能开发是一个持续的过程：

```bash
# 第一轮：整体规划
/ccjk:feat 实现用户评论功能

# 第二轮：细化某个子功能
/ccjk:feat 实现评论的回复功能，支持嵌套回复和@提醒

# 第三轮：优化和增强
/ccjk:feat 优化评论性能，支持分页加载和实时更新
```

## 与其他工作流的区别

| 工作流 | 适用场景 | 特点 |
|--------|---------|------|
| **功能开发工作流** (`/ccjk:feat`) | 新功能规划 | 需求分析、方案设计、UI/UX 规划 |
| **六阶段工作流** (`/ccjk:workflow`) | 具体实施 | 完整开发流程，从研究到评审 |
| **BMad 工作流** (`/ccjk:bmad-init`) | 企业级项目 | 完整团队协作和迭代管理 |

## 注意事项

### Codex 不支持

功能开发工作流目前仅在 Claude Code 中提供。如果使用 Codex，可以使用：

```bash
# Codex 中使用六阶段工作流替代
/prompts:workflow 实现用户评论功能
```

### 代理依赖

功能开发工作流依赖以下代理：

- **Planner**：规划师代理，负责方案设计
- **UI/UX Designer**：UI/UX 设计师代理，负责界面设计

这些代理会在工作流初始化时自动安装。

### 规划结果使用

功能开发工作流输出的规划结果应该：

- ✅ **保存文档**：将规划文档保存到项目目录
- ✅ **版本控制**：纳入 Git 版本控制
- ✅ **团队共享**：与团队成员分享
- ✅ **持续更新**：根据实施情况更新规划
